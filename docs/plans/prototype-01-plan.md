# プロトタイプ 01 計画書

作成日: 2026-04-15  
ステータス: 計画中

---

## 目的

このプロトタイプで検証したいことは **2 つ**に絞る。

1. **Bluesky API の動作確認**  
   ログイン・タイムライン取得・投稿・いいねなど基本操作を実際に動かし、クライアント実装として問題ないことを確認する。

2. **ストレス低減 UI の手触りを確認**  
   調査で明らかになったストレス要因（エンゲージメント数が目に入る・無限スクロール・通知の即時性など）に対する対策を最小限の形で実装し、「実際に使って楽かどうか」を体感する。

---

## スコープ

### 作るもの

| 機能 | 説明 |
|---|---|
| ログイン | App Password によるセッション作成・永続化 |
| タイムライン表示 | フォロー中のホームタイムライン。時系列順（アルゴリズムなし） |
| 投稿 | テキスト投稿（リッチテキスト・メンション・URL 対応） |
| 返信 | スレッド表示と返信送信 |
| いいね | いいね・いいね解除（楽観的更新） |
| リポスト | リポスト・リポスト解除 |
| プロフィール表示 | 基本情報・投稿一覧 |
| 通知 | 通知一覧の取得・表示 |

### ストレス低減機能（プロトタイプで試す対策）

| 対策 | 内容 |
|---|---|
| エンゲージメント数の非表示 | いいね数・リポスト数をデフォルト非表示 |
| 時系列タイムライン | アルゴリズム推薦なし。古い投稿が下に流れていく |
| ページネーション | 無限スクロールを廃止。「もっと読む」ボタンで区切る |
| 通知のバッチ表示 | リアルタイム通知バッジを非表示にし、手動で通知画面を開く方式 |

### 作らないもの（スコープ外）

- 画像・動画投稿（実装コストが高く、本質的な検証に不要）
- OAuth 認証（プロトタイプは App Password で十分）
- フォロー・アンフォロー操作
- ミュート・ブロック
- カスタムフィード切り替え
- 検索
- テキスト・画像のフィルタリング機能（次のプロトタイプ以降）

## 画面構成

```
/ (ログイン画面)
/home (タイムライン)
/notifications (通知)
/profile/:handle (プロフィール)
/post/:uri (投稿詳細・スレッド)
```

レイアウト: 左サイドバー（ナビゲーション）＋ メインコンテンツ（中央カラム）の 2 カラム構成。

## 完成の定義（このプロトタイプが「終わった」と言える条件）

- Bluesky アカウントでログインできる
- タイムラインが表示され、スクロールできる
- いいね・リポスト・テキスト投稿・返信が動作する
- 1 日実際に使ってみて、エンゲージメント数が見えない / 無限スクロールがないことによる体験の変化を感じられる

## 次のプロトタイプに向けた判断軸

このプロトタイプを使った結果として以下のことがわかったら次へ進む。

- 「ページネーション方式は快適か、それとも不便に感じるか」
- 「エンゲージメント数が見えないことで、実際に比較感が減るか」
- 「通知を手動確認にすることで、プレッシャーが減るか、それとも逆に気になるか」
- 「デスクトップアプリとしての操作感・パフォーマンスは問題ないか」
