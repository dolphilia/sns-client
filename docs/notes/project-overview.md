# プロジェクト概要メモ

作成日: 2026-04-15

## 目的

ストレスの原因を極限まで減らしたデスクトップ向け SNS クライアントアプリを作ること。ウェブアプリとしての展開も視野に入れる。

そのために、SNS クライアントに関係する情報・技術・デザインを研究・収集しながら、プロトタイプを繰り返し作成し、最終的なアプリに仕上げていく。

## リポジトリ構成

```
sns-client/
├── docs/
│   ├── notes/    調査メモ・検討メモ
│   ├── plans/    計画書
│   └── spec/     仕様書（草案から始める）
├── prototypes/   プロトタイプ（サブディレクトリごとに管理）
└── research/
    ├── data/       研究用データ
    ├── notebooks/  Jupyter Notebook (.ipynb)
    └── scripts/    ノートブックから呼び出すスクリプト
```

### docs/

- **notes/** — 調査したこと、検討したこと、ちょっとしたメモを置く。このファイルもここに含まれる。
- **plans/** — 開発計画書を置く。
- **spec/** — 仕様書を置く。最初は草案・ドラフトから始め、研究とプロトタイプを通してアプリのイメージが固まるにつれて充実させていく。

### research/

Jupyter Notebook を使って実験・研究を行い、知見をノートにまとめる。対象領域は以下の 2 つ。

- **SNS 関連** — 情報取得・解析・API 調査など
- **ストレス低減手法** — UX 設計・通知設計・情報フィルタリングなど

### prototypes/

プロトタイプごとにサブディレクトリを切る（例: `prototype-a/`, `prototype-b/` など）。

**技術スタック（基本）**

| 項目 | 採用技術 |
|---|---|
| ランタイム | Node.js |
| 言語 | TypeScript |
| ビルドツール | Vite |
| フレームワーク | 必要に応じて React / Next.js などを追加検討 |
| ライブラリ | 積極的に活用する |

## 進め方の方針

1. **研究フェーズ** — `research/` で SNS の情報取得・解析やストレス低減に関する研究を進め、知見を蓄積する。
2. **プロトタイプフェーズ** — `prototypes/` でアイデアを実装し、繰り返し検証する。
3. **仕様策定フェーズ** — 研究とプロトタイプを通じてアプリのイメージが固まったら `docs/spec/` に仕様書を書き始める（草案スタート）。
4. **本開発フェーズ** — 仕様をベースに本格的な開発へ移行する。
