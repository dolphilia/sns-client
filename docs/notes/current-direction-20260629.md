# 現在の開発方針メモ

作成日: 2026-06-29

## 決定

当面の軌道修正として、以下の方針を採用する。

1. デスクトップ基盤は Tauri ではなく Electron へ寄せる。
2. Next.js 化はいったん取り下げる。
3. AI / LLM による投稿の吟味は、処理負荷が重くなりがちなため当面は見送る。
4. 基本コンセプトは「ストレスを減らすデスクトップ SNS クライアント」のまま維持する。
5. ただし、号による体験管理や完了画面など、通常の SNS から大きく離れる仕様はいったん見送る。

## 理由

現在の目的は、SNS クライアント本体の体験を整理し直し、過剰に重い処理や実験的な構成から距離を取ることにある。

Next.js は Web 版やサーバー機能を本格化する段階では再検討余地があるが、現時点のデスクトップ SNS クライアント本体には必須ではない。まずは React + Vite のフロントエンドを活かしつつ、デスクトップ外枠を Electron に移す方が変更範囲を抑えやすい。

AI 吟味は prototype-04 で重要な実験テーマだったが、日常的なクライアント体験の基盤にするには処理時間・ローカル環境依存・モデル管理の重さが大きい。今後は主要導線から外し、必要なら研究・実験ページや将来機能として扱う。

号モデルや「読み終わり」を中心にした設計は prototype-03 / 04 の重要な実験だったが、現時点では一般的な SNS クライアントから離れすぎている。まずは通常の SNS として自然に使える無限フィードへ戻し、その上でストレスを増やしやすい要素だけを抑制する。

## SNS クライアントの体験方針

| 方針 | 現在の扱い |
|---|---|
| 比較を減らす | 継続。いいね数・リポスト数・返信数などのエンゲージメント数はデフォルトで隠す。 |
| 終わりを作る | 撤回。号や完了画面ではなく、一般的な SNS に近い無限フィードへ寄せる。 |
| 自動更新しない | 継続。新着投稿を勝手に差し込まない。ただし、過去の投稿はスクロールで取得して遡れるようにする。 |
| 通知圧を下げる | 継続。リアルタイム通知・バッジ・未読カウントで急かさない。 |
| 交流を最小化する | 継続。返信・リポスト・引用など、負荷の高い交流操作は前面に出しすぎない。 |
| ローカル操作を重視する | 継続。ブックマークなど、相手に通知されない操作を重視する。UI 上でも「相手に通知されない」ことが分かるようにする。 |
| 使い終わりを肯定する | 撤回。専用の完了体験は当面置かず、一般的な SNS のフィード体験へ寄せる。 |
| AI 判定に依存しない | 継続。投稿の表示可否を AI / LLM に依存させない。 |

## 当面残す中核

- Bluesky / AT Protocol 対応のデスクトップ SNS クライアント
- App Password 認証によるプロトタイプ運用
- React + Vite + TypeScript
- TanStack Router
- TanStack Query
- Zustand + persist
- Tailwind CSS + shadcn/ui 系コンポーネント
- エンゲージメント数の非表示
- 一般的な SNS に近いフィード表示
- 明示的な新着取得、またはユーザー操作による更新
- スクロールによる過去投稿の追加取得
- ブックマークなど、相手へ通知しないローカル操作

## 廃止・見送り対象の機能

- 号サイズ設定
- 時間帯に応じた背景色
- 号によるフィード管理
- 読み終わり / 完了画面を中心にした体験
- LLM / AI による投稿の吟味

## 整理対象

- `src-tauri/` と Tauri 依存
- `screeningStore` / `useScreening` / `ollamaFilter` など AI 吟味を前提にした主要導線
- `research.ollama` ルートの扱い
- `@huggingface/transformers` など旧 ML フィルター系依存
- `editionStore` や `components/edition/` など号モデルを前提にした主要導線
- 号サイズ・Ambient Sync など、現方針では廃止する設定
- prototype-03 / 04 計画書に残る、現方針とズレた記述

## 次の実装方針

Electron ベースの `prototype-05` を作成し、AI 吟味・号モデル・時間帯背景を外した通常フィード寄りのクライアントとして検証した。現在はこの実装を正式なデスクトップアプリ本体として `apps/desktop/` に昇格している。

今後は `apps/desktop/` を現行アプリとして扱い、必要な機能追加・整理・検証をこのディレクトリで進める。過去の `prototypes/` は検証履歴として残す。
